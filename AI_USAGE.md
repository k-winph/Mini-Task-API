# การใช้ AI ในโปรเจกต์ Mini Task API

โปรเจกต์ Mini Task API นี้ได้รับความช่วยเหลือจาก AI (Large Language Model) เพื่อเร่งการพัฒนา โดยเฉพาะการวางโครงสร้างเริ่มต้นและการให้ตัวอย่างโค้ดที่นำมาปรับใช้จริงบนฐานโค้ดของทีม

## สิ่งที่ใช้ AI ช่วย
- การสร้างโครงสร้างโปรเจกต์เบื้องต้น รวมถึงการแนะนำ dependencies ที่ควรติดตั้งสำหรับ REST API แบบ Node.js
- การเสนอผังโฟลเดอร์และไฟล์พื้นฐาน เช่น middleware/, routes/, models/, services/ พร้อมตัวอย่างเนื้อหาไฟล์ตั้งต้น
- การร่างโค้ดตัวอย่างสำหรับ middleware ตรวจสอบสิทธิ์, rate limiter, และ task handler ที่นำมาเป็นฐานให้ทีมปรับแต่ง
- การสรุปแนวคิด ABAC/RBAC, โฟลว์ idempotency, และข้อควรระวังด้านความปลอดภัยเพื่อประกอบการออกแบบ
- การให้ตัวอย่างสคริปต์ทดสอบอัตโนมัติและตัวอย่าง request สำหรับ Postman เพื่อย่นเวลาในการเตรียมชุดทดสอบ

## สิ่งที่ทีมพัฒนาเองทั้งหมด
- การออกแบบตรรกะธุรกิจ (Business Logic) สำหรับ ABAC และกระบวนการตรวจสอบสิทธิ์ตามข้อกำหนดของระบบ
- การปรับแต่งโค้ดทั้งหมดให้ทำงานร่วมกับ MySQL จริง รวมถึงการเชื่อมต่อ, migration, และการจัดการทรานแซกชัน
- การสร้างและปรับแต่ง middleware เฉพาะของระบบ เช่น rate limit ต่อผู้ใช้, idempotency key store, และ audit logging
- การทดสอบทุกกรณีด้วย Postman (RBAC, ABAC, rate limit, idempotency) พร้อมการวิเคราะห์ผลและแก้ไขปัญหาที่พบ
- การจัดทำเอกสาร, README, และแนวทาง deployment ให้เหมาะกับสภาพแวดล้อมขององค์กร

## ขอบเขตการใช้งาน AI
- AI ถูกใช้เป็นเครื่องมือช่วยพัฒนา (development assistant) เท่านั้น ไม่ได้ใช้สร้างโปรเจกต์แบบอัตโนมัติทั้งหมด
- ทีมผู้พัฒนาเข้าใจโค้ดทุกส่วนและสามารถอธิบายการทำงานได้ครบถ้วน รวมถึงเหตุผลของทุกการตัดสินใจด้านสถาปัตยกรรม
